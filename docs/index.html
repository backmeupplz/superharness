How it works

Three steps. No new mental model required.

1

Talk to it like you'd talk to opencode

Just describe your task. superharness figures out what can be parallelized and spins up the right number of agents.

2

Agents work in parallel, each in their own workspace

Every agent gets its own git worktree — isolated files, no conflicts. They work simultaneously so you're not waiting for one thing to finish before the next starts.

3

superharness watches them so you don't have to

It auto-approves safe actions, follows up on stalled agents, flags loops, and only interrupts you when something actually needs a human decision.

What you get

Four things that matter.

⚡

Parallel by default

Tasks that can run in parallel, do. Each agent gets its own tmux pane, git worktree, and model. No file conflicts, no waiting.

👁️

Smart supervision

Background monitor detects stalled agents and attempts recovery automatically. Loop detection catches agents spinning in circles. You only get interrupted when it matters.

🌐

Works with any model

100+ models across Anthropic, OpenAI, Fireworks, OpenRouter, Cloudflare AI Gateway, and more. Pick a different model per agent if you want.

🔬

Plan + Build modes

Agents can run read-only (plan mode) to analyze before touching anything, or build mode for full file access. You choose per agent.

If you know opencode, you already know this.

superharness uses opencode under the hood. Everything you know still applies — it just adds one layer on top.

opencode
  • One AI agent per session
  • Single working directory
  • Sequential task execution
  • You watch one terminal
superharness
  • Multiple agents running simultaneously
  • Isolated worktree per agent
  • Parallel task execution
  • Automated supervision and recovery

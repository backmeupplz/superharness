SuperHarness gives you a complete toolkit for running AI coding agents as a coordinated team.

⚡

Parallel Spawning

Launch any number of AI coding agents simultaneously, each isolated in its own tmux pane and git worktree. No conflicts, no drama.

🎭

Plan + Build Modes

Plan-mode agents explore and propose (read-only, blue border). Build-mode agents implement (full access, green border). Always review before you build.

🔁

Loop Detection

Automatic detection of stuck agents repeating the same action. loop-status spots oscillation patterns; loop-clear resets history after intervention.

🌙

Away Mode

Step away without losing control. Queue decisions for uncertain situations, let safe work continue, and review a full decision log when you return.

💓

Autonomous Monitor

Background daemon polls every N seconds, detects stalled workers, and attempts auto-recovery (Enter → "continue" → escalate to human).

🎛️

Pane Management

Hide workers to background tabs, surface them on demand, resize, apply layout presets (tiled, main-vertical…), and never lose context.

🌐

100+ AI Models

Works with Anthropic, OpenRouter, Fireworks AI, Cloudflare AI Gateway and more. Pick the right model for each worker independently.

🔐

Permission Guard

Workers ask before running risky commands. You approve safe ops, deny destructive ones, and escalate ambiguous calls to the human in the loop.

🌿

Git Worktrees

Each worker gets its own isolated git worktree — parallel branches, no file conflicts, clean merges when work is done.

Two modes. One workflow.

Use plan mode to think, build mode to act. The recommended flow prevents wasted implementation work.

Plan

Read-Only Exploration

The agent analyzes the codebase, reads files, searches for patterns, and produces a written plan — without touching a single file. Blue pane border. Use this first.

--mode plan
Build

Full Implementation Access

The agent can create, edit, delete files and execute commands freely. Green pane border. Feed it the plan from a plan-mode agent for best results.

--mode build  (default)
plan → build workflow
# Step 1: Let a plan-mode agent explore $ git worktree add /tmp/plan-1 HEAD $ superharness spawn \ --task "Analyze the payment module and propose a refactor" \ --dir /tmp/plan-1 \ --mode plan \ --model openrouter/google/gemini-2.5-pro ✓ Spawned pane %2 [plan] — blue border # Step 2: Read the plan, then implement $ superharness read --pane %2 --lines 60 [Plan output: split PaymentService into 3 modules...] $ git worktree add /tmp/build-1 HEAD $ superharness spawn \ --task "Implement refactor: split PaymentService into Auth, Charge, Refund modules" \ --dir /tmp/build-1 \ --mode build ✓ Spawned pane %3 [build] — green border

How it works

SuperHarness is the conductor. You set the direction. Workers handle the code.

1

Decompose the task

Break your goal into independent subtasks that can run concurrently without file conflicts. Good decomposition = maximum parallelism.

2

Create git worktrees

Run git worktree add /tmp/worker-N HEAD for each planned worker. Isolated filesystem, isolated branch — parallel work without collisions.

3

Spawn agents

Call superharness spawn --task "..." --dir /tmp/worker-N --model MODEL. Each agent runs in its own tmux pane with a clear task context and chosen AI model.

4

Poll & respond

Every 30–60s, superharness read --pane %ID to check progress. Workers ask permission for risky actions — approve, deny, or escalate. Answer questions with send.

5

Monitor autonomously

Start superharness monitor for background health checks. Stalled workers get auto-recovery attempts. Loops get flagged. You only see what needs human attention.

6

Collect & merge

When workers finish, kill the pane, cherry-pick or merge the worktree branch, remove the worktree. Clean results, ready to review.

Quick Start

Up and running in under two minutes.

1 · Install
$ cargo install superharness Compiling superharness v0.2.0 Finished in 4.2s Installed ~/.cargo/bin/superharness
2 · Initialize your repo
# In your project directory: $ superharness init ✓ Created AGENTS.md ✓ tmux session "superharness" ready
3 · Spawn your first agent
$ git worktree add /tmp/w1 HEAD $ superharness spawn \ --task "Add dark mode to the dashboard" \ --dir /tmp/w1 \ --mode build \ --model anthropic/claude-sonnet-4-6 ✓ Spawned pane %2 [build]
4 · Monitor & interact
$ superharness read --pane %2 --lines 30 $ superharness send --pane %2 --text "y" $ superharness monitor --interval 45 → Watching all panes...

Step away. Come back informed.

Away mode queues decisions you'd normally need to be present for, so workers keep making safe progress while you're gone.

away mode workflow
# Before leaving: $ superharness away --message "Back in 2 hours" ✓ Away mode enabled. Safe work continues; decisions queued. # Worker asks something uncertain — queued instead of auto-approved: ⏳ Queued: "Should I refactor module X into Y?" (pane %4) # When you return: $ superharness present ✓ Present mode. Pending decisions: 1. [%4] Refactor module X into Y? — context: breaks existing API 2. [%5] Use tokio or async-std? — context: tokio has wider ecosystem $ superharness send --pane %4 --text "Yes, refactor it" $ superharness send --pane %5 --text "Use tokio" $ superharness clear-decisions ✓ All decisions resolved.
